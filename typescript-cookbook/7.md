## 1. Git 인증 설정

### Git 사용자 정보 설정

```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

macOS credential helper 설정

git config --global credential.helper osxkeychain

---
2. GitHub CLI 설치 및 인증 (PR 생성용)

GitHub CLI 설치

brew install gh

GitHub 인증 (웹 브라우저로 로그인)

gh auth login

인증 과정:

1. Where do you use GitHub? → GitHub.com 선택
2. What is your preferred protocol for Git operations? → SSH 선택 (권장)
3. Upload your SSH public key? → SSH 키 경로 선택
  - 예: /Users/raehan/.ssh/id_ed25519_work_github_20250306.pub
4. Title for your SSH key → 원하는 이름 입력
  - 예: claude_code
5. How would you like to authenticate? → Login with a web browser 선택
6. Press Enter → 브라우저에서 인증 완료

성공 메시지:

✓ Authentication complete.
✓ Configured git protocol
✓ SSH key already existed on your GitHub account
✓ Logged in as Raehan989

인증 확인

gh auth status

예상 출력:

github.com
  ✓ Logged in to github.com account Raehan989 (keyring)
  - Active account: true
  - Git operations protocol: ssh
  - Token: gho_************************************
  - Token scopes: 'admin:public_key', 'gist', 'read:org', 'repo'

3. 토큰 권한 확인

GitHub CLI 인증 시 다음 권한이 필요합니다:

| 권한               | 용도                      |
|------------------|-------------------------|
| repo             | 저장소 접근 (PR 생성, 코드 푸시)   |
| workflow         | GitHub Actions 워크플로우 관리 |
| write:discussion | PR/Issue 코멘트 작성         |

권한이 부족한 경우

gh auth refresh --scopes repo,workflow,write:discussion
```
